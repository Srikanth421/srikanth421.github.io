01 — Weight Initialization

Xavier & He Initialization

Before training starts, weights must be set carefully. Too large and activations explode. Too small and gradients vanish before they reach early layers. Both kill learning before it begins.

Xavier (Glorot) — for sigmoid / tanh

Keeps the variance of activations equal across layers when using symmetric activations like sigmoid or tanh.

W ~ Uniform[ -√6 / √(n_in + n_out), +√6 / √(n_in + n_out) ] // Our network: n_in=3 inputs, n_out=4 hidden neurons Limit = √6 / √(3+4) = 2.449 / 2.646 = ±0.926 Weights sampled from Uniform[ -0.926, +0.926 ]

He Initialization — for ReLU

ReLU kills negative inputs (outputs 0), so roughly half the neurons are inactive at any moment. He doubles the variance to compensate, keeping the signal alive through deep networks.

W ~ Normal( 0, √(2 / n_in) ) // Our fraud network: n_in = 3 features std = √(2/3) = 0.816 Example weights drawn from N(0, 0.816²): W₁ = 0.63 W₂ = -1.12 W₃ = 0.44
Dying ReLU problem: If weights start too large, neurons receive very negative inputs, output 0, and get zero gradient — they never update. Like an employee given an impossible task on day one who gives up entirely. He init sets weights small enough that most neurons start active.

02 — Batch Normalization

Batch Normalization

Transaction amounts range from $1 to $50,000. Without normalization, a single large feature value dominates the neuron's signal. Batch Norm re-centers and re-scales activations at every layer, every mini-batch.

Think of it as: Adjusting the volume knob after every song in a playlist so each track plays at the same loudness — you can hear the details regardless of how loud the original recording was.

Step-by-step calculation

Given raw hidden activations for a mini-batch of 3 fraud transactions: z = [2.1, −0.4, 1.3]

1
Compute mean
μ = (2.1 + −0.4 + 1.3) / 3
μ = 1.0
2
Compute variance
σ² = [(2.1−1)² + (−0.4−1)² + (1.3−1)²] / 3
= [1.21 + 1.96 + 0.09] / 3
σ² = 1.087, σ = 1.043
3
Normalize each value
ẑ = (z − μ) / σ
ẑ = [1.007, −1.342, 0.288]
4
Scale & shift (learned γ=1.2, β=0.1)
ŷ = γ · ẑ + β
ŷ = [1.308, −1.510, 0.446]
// Full batch norm formula ẑ = (z − μ) / √(σ² + ε) // ε prevents division by zero ŷ = γ · ẑ + β // γ, β are learned per neuron

γ and β are trainable parameters — the network learns the best scale and offset for each neuron, rather than forcing every layer to have exactly mean=0, std=1.

03 — Dropout

Dropout

During training, randomly zero out neurons with probability p. This prevents co-adaptation — where neurons become so reliant on each other that the network memorizes training transactions instead of learning general fraud patterns.

Analogy: A study group where one person writes all answers and others copy. Great for that exam, useless for any new one. Dropout is the teacher randomly removing group members each session — everyone learns independently and becomes more robust.

04 — SGD with Momentum

SGD with Momentum

Stochastic means we use a random sample (mini-batch of ~32 transactions) rather than the entire dataset. This makes each gradient estimate noisy — but we take thousands of steps where classical gradient descent would take one.

Momentum accumulates a velocity vector from past gradient steps. Like a bowling ball rolling downhill — it ignores small pebbles (noisy gradients) and keeps rolling in the main direction.

β=0.9 means: "Keep 90% of my previous direction (velocity), add 10% of today's gradient." Consistent gradient directions build up speed. Opposing directions cancel out. The result: faster, smoother convergence on noisy fraud data.

Plain SGD vs SGD + Momentum — side by side

// Plain SGD W_new = W − α · ∇L // SGD with Momentum v_new = β · v_prev + ∇L // accumulate velocity W_new = W − α · v_new

05 — Adam Optimizer

Adam Optimizer

Adam combines two ideas: Momentum (1st moment — tracks gradient direction) and RMSProp (2nd moment — tracks gradient magnitude). The result is an adaptive learning rate per weight, with bias correction for the cold start problem.

Adaptive learning rate: A GPS that recalculates speed per road type — highway gets fast moves, school zones get careful small steps. Adam does this per weight, per update. In fraud detection, transaction_amount (huge values) and time_of_day (small values) each get appropriate step sizes automatically.

Bias correction — why it matters

The moment estimates m and v start at 0. Early in training this pulls them too close to zero. Dividing by (1−βᵗ) corrects this — at t=1 with β₁=0.9, we divide by 0.1, scaling up by 10× to restore the true magnitude.

// 1st moment — momentum direction m_t = β₁ · m_{t-1} + (1 − β₁) · g_t // 2nd moment — squared gradient (adaptive scale) v_t = β₂ · v_{t-1} + (1 − β₂) · g_t² // Bias correction (fixes cold start at t=0) m̂_t = m_t / (1 − β₁ᵗ) v̂_t = v_t / (1 − β₂ᵗ) // Final weight update W_{t+1} = W_t − α / (√v̂_t + ε) · m̂_t

07 — Visual Intuition

Optimizer paths on the loss surface

Imagine the loss landscape as a hilly terrain. Each optimizer traces a different path toward the minimum (lowest loss).

Plain SGD zig-zags because noisy mini-batch gradients pull in slightly different directions each step. Momentum smooths this out. Adam additionally adapts the step size per weight, taking a nearly direct path.

08 — Summary

Putting it all together

How all five techniques combine in a production fraud detector:

Technique What it does Why it matters for fraud
He init W ~ N(0, √2/n_in) ReLU neurons don't die at start; rare fraud patterns get a signal from step 1
Xavier init W ~ U[±√6/√(n_in+n_out)] Use for sigmoid/tanh; keeps gradients from exploding or vanishing
Batch Norm Normalize → γ·ẑ + β per batch Transaction amounts span $1–$50k; BN keeps activations stable across that range
Dropout Zero p% of neurons per step Forces neurons to learn independently; prevents memorizing specific fraud patterns
SGD + Momentum v = β·v + ∇L; W -= α·v Smooths noisy 99:1 class-imbalanced gradients; faster convergence
Adam Adaptive lr via 1st + 2nd moments Best default; handles different scales of amount, time, distance features automatically

Recommended setup

He initialization Batch Norm (after each hidden layer) Dropout p=0.3 Adam α=0.001 Weighted loss (class imbalance)
