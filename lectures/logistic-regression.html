01
The 4 Formulas — Memorise These
Formula 01
Sigmoid Function
Squashes any real number → probability in (0, 1)
σ(z) = 1 / (1 + e^(-z)) σ(-∞) → 0 σ(0) = 0.5 σ(+∞) → 1
Formula 02
Forward Pass
Linear combination → sigmoid → probability
z = X · w + b ŷ = σ(z) = σ(X · w + b)
Formula 03
Binary Cross-Entropy Loss
Penalises confident wrong predictions exponentially
L = -1/n · Σ [ y · log(ŷ) + (1-y) · log(1-ŷ) ] y=1, ŷ≈1 → loss ≈ 0 ✓ y=1, ŷ≈0 → loss → ∞ ✗
Formula 04
Gradients + Weight Update
Chain rule result — log + sigmoid derivatives cancel cleanly
∂L/∂w = X.T · (ŷ - y) / n ∂L/∂b = mean(ŷ - y) w ← w - lr · ∂L/∂w b ← b - lr · ∂L/∂b

Complete Pseudocode
LogisticRegression.pseudo
# Hyperparameters self.lr = learning_rate # step size, e.g. 0.01 self.epochs = epochs # max iterations, e.g. 1000 self.tolerance = tolerance # early stop threshold, e.g. 1e-4 # Learned parameters (set during fit) self.weights = None # shape: (n_features,) self.bias = None # scalar # Early stopping tracker self.prev_loss = infinity # start high so first epoch always runs
# Convert raw score → probability RETURN 1 / (1 + e^(-z)) # Works element-wise on arrays # sigmoid(-∞) → 0 | sigmoid(0) = 0.5 | sigmoid(+∞) → 1
# y = actual labels (0 or 1) shape: (n,) # y_hat = predicted probabilities shape: (n,) loss = -mean( y * log(y_hat) + # term for class 1 (1-y) * log(1 - y_hat) # term for class 0 ) RETURN loss # single scalar — lower is better # Used to: monitor training, trigger early stopping # NOT directly used in weight updates (gradients are)
── INITIALISE ────────────────────────────────────── weights ← zeros(n_features) # X.shape[1] bias ← 0 ── TRAINING LOOP ─────────────────────────────────── FOR epoch IN range(epochs): ┌── FORWARD PASS ───────────────────────────────┐ z = X · weights + bias # linear step y_hat = sigmoid(z) # probabilities └───────────────────────────────────────────────┘ ┌── LOSS ───────────────────────────────────────┐ loss = binary_cross_entropy(y, y_hat) └───────────────────────────────────────────────┘ ┌── EARLY STOPPING ─────────────────────────────┐ IF |prev_loss - loss| < tolerance: PRINT "Converged at epoch", epoch BREAK prev_loss ← loss └───────────────────────────────────────────────┘ ┌── BACKWARD PASS (gradients) ──────────────────┐ grad_w = X.T · (y_hat - y) / n # shape: (n_features,) grad_b = mean(y_hat - y) # scalar └───────────────────────────────────────────────┘ ┌── UPDATE WEIGHTS ─────────────────────────────┐ weights ← weights - lr · grad_w bias ← bias - lr · grad_b └───────────────────────────────────────────────┘
# Just the forward pass — no learning z = X · weights + bias RETURN sigmoid(z) # probabilities in (0, 1)
probs = predict_proba(X) RETURN 1 if prob > threshold else 0 # per sample # In numpy (one line): # (probs > threshold).astype(int)
03
Why Each Piece Exists
Component Why it exists
sigmoid(z)Neural networks need outputs as probabilities. Sigmoid maps any real number to (0,1) — making the output interpretable as a probability.
X · w + bLearn a weighted combination of features. The bias b lets the decision boundary shift freely, not just pass through the origin.
binary_cross_entropyPenalises confident wrong predictions exponentially. Pairs perfectly with sigmoid — their derivatives cancel cleanly in the chain rule.
X.T in grad_wShape alignment. X is (n, f), error is (n,). X.T · error gives (f,) — exactly one gradient per weight. Without .T the shapes don't multiply.
(ŷ - y) error termLarge when wrong, small when right. Drives weight updates proportionally to how wrong the prediction was.
learning rate (lr)Controls step size. Too large = overshoot and diverge. Too small = very slow convergence. Typically 0.01 or tuned via grid search.
early stoppingStop when loss improvement < tolerance. Prevents wasted computation and reduces overfitting on training data.
weights = zerosSafe initialisation. Unlike deep nets, logistic regression doesn't suffer from symmetry breaking issues with zero init.

Connection to Neural Networks

Logistic regression IS a neural network — one neuron, no hidden layers. Everything here scales to deep learning.

Logistic Regression Neural Network Equivalent
sigmoid(z)→activation function (ReLU, tanh, softmax...)
X · w + b→fully connected layer (nn.Linear)
binary_cross_entropy→loss function (cross-entropy, MSE, focal...)
X.T · (ŷ-y) / n→backpropagation (chain rule across all layers)
weight update loop→optimizer (SGD, Adam, RMSProp...)
one neuron→millions of neurons across many layers
early stopping→early stopping + learning rate scheduler
05
Interview One-Liners
On Sigmoid
Sigmoid squashes any real number into a probability between 0 and 1. It's the activation function that turns logistic regression into a classifier rather than a regressor.
On Binary Cross-Entropy
Binary cross-entropy penalises confident wrong predictions exponentially. It's paired with sigmoid because their derivatives cancel cleanly in the chain rule, giving us the simple (ŷ - y) error term.
On the Gradient Formula
The gradient X.T dot (ŷ - y) / n comes from applying the chain rule to the loss. X.T is needed purely for shape alignment — we need one gradient per feature, not per sample.
On Early Stopping
Instead of blindly running all epochs, we track the loss each iteration and stop when the improvement drops below a tolerance threshold. This saves compute and reduces overfitting.
On the Neural Network Connection
Logistic regression is a single-neuron neural network. The forward pass is identical to one linear layer followed by sigmoid. Backprop here is just the chain rule across one layer — the simplest possible case.
