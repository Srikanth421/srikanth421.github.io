Why Were Transformers Needed? The problem with what you already know

⚙ The Core Problem

Word2Vec, GloVe, and FastText produce static word embeddings. Each word gets one fixed vector — but to represent a whole sentence, you average all vectors together.

Averaging is commutative:

A + B + C  = same result as
C + B + A

So these two sentences produce identical vectors:

dogbitesman→ avg = [0.55, 0.61]
manbitesdog→ avg = [0.55, 0.61]
🚫 The model cannot tell who bit whom. Word order is invisible.

Also: Word2Vec gives "bank" the same vector in "river bank" and "bank loan". Context is ignored completely.

💡 Analogy

A Bag of Scrabble Tiles

Imagine tipping the words of a sentence into a bag and shaking it up. You lose all sense of order.

"The chef cooked the fish" and "The fish cooked the chef" become the same bag — same tiles, different meaning, different meal.

Word2Vec is that bag. Rich word meaning, zero sentence structure.

💡 This is the gap Transformers were designed to fill: meaning + order + context, all at once.

Transformer Overview The big picture

⚙ What Is a Transformer?

A Transformer is a neural network architecture built entirely on attention mechanisms — no recurrence, no convolutions.

It has two main parts:

  • Encoder — reads and understands the input
  • Decoder — generates the output
📥 Input Sentence
↓
🔡 Word Embeddings + Position
↓
🧠 Encoder (understand)
↓
💡 Context Vector
↓
✍ Decoder (generate)
↓
📤 Output Sentence
💡 Analogy

The Human Translator

Imagine a professional translator working on a Spanish document:

  1. Read the full English sentence, holding the entire meaning in mind
  2. Understand not just words, but tone, context, and relationships
  3. Generate the Spanish translation one word at a time, referencing understanding
Input (English):
"The cat sat on the wall"
Output (Spanish):
"El gato se sentó en la pared"
Key insight: the translator doesn't just look up each word. They understand the whole sentence first — that's what the Encoder does.

Positional Encoding Teaching the model word order

⚙ Technical

Attention mechanisms process all words simultaneously — they have no built-in sense of order. Positional Encoding fixes this by adding a unique position signal to each word's embedding.

PE(pos, 2i)  = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

// d = embedding dimension, pos = word position

The final input to the transformer is:

Input = Word Embedding + Positional Encoding

Now "dog" at position 1 has a different combined vector than "dog" at position 3. Order is preserved.

Self-Attention Every word looks at every other word

⚙ Technical

In self-attention, each word computes a weighted relationship with every other word in the sentence — including itself. The output is a context-aware representation.

Example sentence:

Thecatsat onthewall becauseitwastired

Question: What does "it" refer to?

Self-attention calculates how much each word should influence the meaning of "it". The model learns that cat is most relevant:

💡 Analogy

The Detective Solving a Mystery

A great detective doesn't hear one clue in isolation. When they encounter the word "it", they immediately scan every previous clue in the case file looking for what "it" refers to.

🔍 Detective's Notebook:
Clue 1 — "The cat"  →  Strong lead ★★★
Clue 2 — "the wall"  →  Possible ★★
Clue 3 — "sat, on, the"  →  Weak ★
Conclusion: "it" = the cat (70% confidence)

Self-attention is that detective. For every word, it scores how relevant every other word is — then uses those scores to build a richer, context-aware understanding.

💡 Key insight: Word2Vec gives "it" a single fixed vector. Self-attention gives "it" a different vector in every sentence, depending on context.

Query · Key · Value The mechanism behind self-attention

⚙ Technical
Query (Q)
❓ Seeker

What am I looking for?
Current word's question

Key (K)
🏷 Label

What do I contain?
Every word's identity tag

Value (V)
📦 Content

What can I provide?
The actual information

The attention flow:

Query × Key → Raw Score
↓
÷ √d (scale)
↓
Softmax → Attention Weights
↓
Weights × Values
↓
Context Vector Output
Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V

Attention Calculation A numerical worked example

⚙ Step-by-Step Calculation
Step 1 — Inputs
Query = [1, 2]

Key₁ = [1, 1] ← Word 1
Key₂ = [3, 4] ← Word 2
Step 2 — Dot Products
Q · K₁ = (1×1) + (2×1) = 3
Q · K₂ = (1×3) + (2×4) = 11

// Divide by √d_k = √2 ≈ 1.41
Score₁ = 3 / 1.41 = 2.12
Score₂ = 11 / 1.41 = 7.80
Step 3 — Softmax
e^2.12 = 8.33
e^7.80 = 2440.6
Total = 2448.9

Weight₁ = 8.33 / 2448.9 = 0.003
Weight₂ = 2440 / 2448.9 = 0.997
Step 4 — Weighted Values
Value₁ = [1, 0] × 0.003 = [0.003, 0]
Value₂ = [0, 1] × 0.997 = [0, 0.997]

// Sum = final output
Output = [0.003, 0.997]

// Word 2 dominated! (7.80 >> 2.12)
📊 What this means: The Query was most similar to Key₂ (dot product 11 vs 3), so the output is almost entirely shaped by Value₂. Softmax sharpens this into near-certainty.

Multi-Head Attention Many perspectives at once

⚙ Technical

Instead of one attention computation, run multiple attention heads in parallel, each learning different relationships.

h = 8 heads    // typical in base Transformer
d = 512 total dimensions
d_head = 512 / 8 = 64 per head

Output = Concat(head₁, …, head₈) × W°
= [64×8] = 512 dimensions

Each head independently learns to attend to different patterns:

The Crime Investigation Team

One detective might miss something. A great police department sends a whole team, each specializing in a different angle:

🔵

🟢

After each detective files their report, the chief combines all findings into one comprehensive understanding. That's concatenation + projection.

💡 One attention head would make one guess. Eight heads together build a 360° understanding of every sentence.

Encoder Block How understanding is built

⚙ Architecture

One encoder block repeats N times (N=6 in original Transformer). Each pass deepens understanding:

Encoder Block (×6)
📥 Input Embeddings + Position
↓
🧠 Multi-Head Self-Attention
↓
➕ Add & Norm (Residual)
↓
⚡ Feed-Forward Network
↓
➕ Add & Norm (Residual)
↓
📤 Enriched Representation

Key components explained:

Residual Connection (Add): Output = Layer(x) + x. Passes original input alongside transformed version — prevents gradient vanishing.
Layer Norm: Normalises values to have mean≈0, std≈1. Keeps training stable.
Feed-Forward Network: Two linear layers with ReLU. Adds non-linearity — the model can learn complex patterns.
💡 Analogy

Reading a Novel Six Times

Imagine reading a complex novel once. You get the gist. Read it again — you notice foreshadowing. Read it a third time — themes emerge.

Each encoder pass is like another careful reading of the sentence. Relationships get richer, patterns get clearer, understanding gets deeper.

Residual connection = you never forget what you read before. Each pass adds to — not replaces — your understanding.

Pass 1: basic word meanings
Pass 2: sentence structure
Pass 3: pronoun resolution
Pass 4: tone and sentiment
Pass 5: topic and domain
Pass 6: nuance and implication
✅ After 6 passes, "it was tired" is fully resolved to "the cat was tired" — the model has built a complete understanding.

Decoder Block Generating the output

⚙ Architecture

The decoder has three sub-layers instead of two, and generates output one token at a time:

Decoder Block (×6)
🔒 Masked Multi-Head Self-Attention
↓
➕ Add & Norm
↓
🔗 Cross-Attention (Encoder ↔ Decoder)
↓
➕ Add & Norm
↓
⚡ Feed-Forward Network
↓
➕ Add & Norm
↓
🎯 Linear + Softmax → Next Word
Cross-Attention: The decoder's Query comes from the decoder, but Keys and Values come from the encoder output. This is how the decoder "reads" the encoder's understanding.
💡 Analogy

Writing an Essay One Word at a Time

Imagine you've just read a detailed brief (encoder's job). Now you must write your response — but with one strict rule: you can only look at what you've already written, never at future words.

Step 1: [START] → writes "El"
Step 2: El → writes "gato"
Step 3: El gato → writes "se"
Step 4: El gato se → writes "sentó"
Step 5: El gato se sentó → writes "en"
…

At every step the decoder:

  1. Looks at its own words so far (masked self-attention)
  2. Looks at the encoder's full understanding (cross-attention)
  3. Predicts the next best word

Masking Preventing the decoder from cheating

⚙ Technical

During training, the decoder sees the full target sentence — but it must not use future tokens to predict the current token. Masking enforces this.

Green ✓ = allowed to attend  |  Red ✗ = masked (set to −∞ before softmax)

Setting future scores to −∞ before softmax means e^(−∞) = 0 — those positions get exactly zero attention weight.
💡 Analogy

The Covered Answer Sheet

Imagine sitting an exam where the questions are printed below your answer box. The invigilator places a card over all the questions you haven't reached yet.

When answering question 3, you can:

  • ✅ See questions 1 and 2 (and your own answers)
  • ❌ NOT see questions 4, 5, 6… (covered!)
Predicting "love" → can see: "I" ✓
Predicting "football" → can see: "I love" ✓
Cannot peek at "football" while predicting "love" ✗
💡 Why mask? So the model genuinely learns to predict — if it could see the answer, it would just copy rather than learn patterns.

At inference time (real use), masking is natural — future words don't exist yet anyway!

Feature 🔵 BERT 🟣 GPT
ArchitectureEncoder-onlyDecoder-only
Reads directionBidirectional ↔Left-to-right →
Primary strengthUnderstanding textGenerating text

Sees full sentence?Yes (both directions)No (left context only)

🔵 Use BERT when: You need to understand or classify text — sentiment analysis, extracting answers from documents, named entity recognition.
🟣 Use GPT when: You need to generate text — chatbots, code generation, summarization, translation, creative writing.
