Technical View

The AI model is not “inside the webpage.” The browser sends an HTTPS request. A backend service validates it, gathers context, applies business logic, may call a model, and returns HTML or JSON.

In CanadianBankNews, Next.js runs on Cloud Run while Vertex AI handles selected AI tasks. Cached public content may be served before Cloud Run is needed at all.

🎭 Stage Analogy

The user sees the actor deliver one line. Behind the curtain are lighting, sound, stage management, costume, and timing.

The AI model is an important performer, but it is not the entire theatre.

Teaching idea: deployment is not finished when a model endpoint responds. Production begins when the endpoint is connected safely to users, data, business rules, caching, observability, and rollback.

Latency Is Additive

If tasks run sequentially, the user experiences roughly the sum of their delays. One slow external source can hold the entire response hostage.

A prototype can tolerate this. A production site with mobile users, crawlers, and repeat visitors cannot.

Engineering Insight

The first performance question is not “Which server is faster?”

It is: Which work should not happen during this request at all?

Module 3 · The Highest-Leverage Change

Move Expensive AI Work Out of the Request Path

AI summaries and slow-changing rate snapshots can be prepared before a visitor asks for them.

Before vs after

❌ Before: Cook While the User Waits

User opens homepage
Fetch external content
Call Vertex AI
Build summary
Render and return
→

✅ After: Prepare Ahead

Scheduled job runs
Fetch news and rates
Generate summaries
Store results
User reads cached result

Technical Pattern

This is precomputation. A scheduled process performs expensive or slow work outside the live request. The website then reads the prepared result.

The pattern is especially useful when content changes every few hours or once per day, rather than every second.

🍲 Restaurant Analogy

A restaurant does not grow vegetables after you order soup. It prepares stock before opening, then completes the final steps when the order arrives.

Scheduled AI work is the digital kitchen prep.

Key principle: AI should be in the critical path only when the response truly requires fresh, user-specific generation.

Why TTFB Drops

Time to First Byte measures how long the browser waits before the first response byte arrives. A cache hit can avoid application startup, external requests, database work, and rendering.

📚 Library Analogy

The origin server is the archive room. The CDN is the popular-books shelf near the entrance.

Readers get common material quickly, while rare or private requests still go to the archive.

Browser caching helps one visitor. CDN caching can help many visitors and crawlers. It also reduces load on Cloud Run and creates a shared edge layer between the public internet and the application.
Module 6 · Next.js Rendering Choices

Do Not Render Every Route the Same Way

A bank guide, a live news feed, and an authenticated dashboard have different freshness and privacy requirements.

Server RenderingRequest-dependent public pagesFresh response, more origin work
Dynamic / No-storeAuthenticated and personalized flowsCorrect privacy, little shared caching
📘

Good ISR Candidate

Mortgage education guide, bank profile, credit card explainer, interview-prep landing page.

⚡

Good Dynamic Candidate

Live user session, saved scenarios, admin dashboard, request-specific API.

🧩

Hybrid Page

Static educational shell with a dynamic calculator or advisor loaded separately.

Architecture is classification: first classify the route by freshness, privacy, and reuse. Then choose its rendering mode.

Database Guidance

Databases remain essential for users, saved scenarios, permissions, and durable product data. But a public guide should not perform repeated joins merely to render identical content for every visitor.

Reduce unnecessary queries, batch lookups, index frequent access paths, and precompute slowly changing aggregates.

🚧 Toll Booth Analogy

The database should support the road network, not become a toll booth every visitor must stop at.

Use it when state is needed. Bypass it when the response is already safely prepared.

Production Boundary

The browser should never directly call the model with trusted credentials. Cloud Run validates the request, chooses tools and context, applies limits, and controls what is returned.

For financial calculations, deterministic code produces the numbers. The model can help explain them in natural language.

Production performance checklist

✓ Move expensive AI work outside the request path where freshness permits.
✓ Cache only visitor-independent content.
✓ Keep authenticated and personalized responses private or no-store.
✓ Use Cloud CDN only after origin cache rules are correct.
✓ Match Next.js rendering mode to each route.
✓ Avoid external live fetches during first render where possible.
✓ Measure TTFB and user-facing Core Web Vitals.
✓ Monitor logs and errors after every deployment.
✓ Keep Dev and Production infrastructure isolated.
