Foundational concepts — visual explanations

Logistic regression & the neuron

A single neuron takes inputs, multiplies each by a weight, sums them, adds a bias, then squashes the result through a sigmoid to get a probability between 0 and 1.

A deep network is just many of these stacked together — each layer's outputs become the next layer's inputs.

"A deep network is many logistic-regression computations stacked together."

Epochs

One epoch = one complete pass through every training example. The model sees the whole dataset, computes loss, and updates weights.

Training for more epochs gives the model more chances to improve — like re-reading a textbook. But too many epochs leads to overfitting.

"Like reading the whole textbook once — cover to cover."

SGD — stochastic gradient descent

The model lives on a hilly loss landscape. It needs to reach the lowest valley. At each step it measures the slope (gradient) and takes a small step downhill.

The learning rate controls step size — too large and it overshoots the valley; too small and it crawls.

"Hiking down a foggy mountain, feeling your way to the lowest point."

Loss function

After each prediction, the loss function compares the model's output to the true label and returns a single number — how wrong was it?

The model's entire goal during training is to drive this number down. Binary Crossentropy is standard for fraud detection.

"The model's pain meter — lower is better."

Representation learning

Each layer transforms data into a more useful form. The network builds its own abstract language for describing fraud — no human designed these features.

Raw numbers → learned patterns → risk signals → probability.

"Not just prediction learning. Representation learning."

Parameters — the model's memory

Every connection between neurons stores a weight. Every neuron also has a bias. These are the only numbers that change during training.

For 7 inputs → 128 neurons: 7×128 + 128 = 1,024 parameters in that layer alone.

"Parameters are the memory of the neural network."

Deep learning vs machine learning

Machine Learning
Requires manual feature extraction
Less effective on unstructured data
Lower computational requirements
Generally more interpretable (white box)
Deep Learning
Automatic feature extraction
Excels with images, text, audio
Needs significantly more compute & memory
Often a "black box" — less interpretable
"Deep learning is not just prediction learning. It is representation learning."

Architecture examples — how depth and width change the network

Depth (more layers)
Learns complex, abstract patterns
Each layer builds on the previous. Early layers detect edges; later layers detect shapes and concepts.
Width (more neurons)
More patterns learned in parallel
A wider layer can represent more features at once, but adds parameters without adding abstraction.
Trade-off
More parameters = more memory, more risk
Bigger networks train slower and can overfit. The right architecture depends on the problem and data size.

Activation functions

Sigmoid

1 / (1 + e⁻ˣ)
  • Output: 0 → 1
  • Use in binary output layer
  • Prone to vanishing gradients

ReLU

max(0, x)
  • Output: 0 → ∞
  • Fast for hidden layers
  • Can "die" for negative inputs

Tanh

(1−e⁻²ˣ)/(1+e⁻²ˣ)
  • Output: −1 → +1
  • Zero-centred
  • Handles negatives better

Leaky ReLU

max(αx, x)
  • Fixes dying ReLU
  • Small slope for x < 0
  • Good general-purpose choice
