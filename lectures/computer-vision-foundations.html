Module 1 · Introduction

What Is Computer Vision?

Teaching a machine to "see" is really teaching it to turn pixels into decisions — the same way your eyes turn light into "duck it, that's a ball."

Technical Definition

Computer Vision is the branch of AI that lets machines derive meaningful information from digital images, video, and other visual input — and then act on that information.

The typical pipeline looks like this:

🔦 Analogy

Think of walking into a dark room and feeling for a light switch. You don't consciously reason "this is a rectangular plastic protrusion at hand height" — your brain does that instantly and jumps straight to "flip it."

A CV model does the same, except its "feeling around" is entirely numerical: every image is just a giant spreadsheet of numbers, and the model learns which patterns of numbers correspond to "switch," "cat," or "stop sign."

Module 2 · Representation

Pixels, Color Spaces & Image Arrays

Before a model can find a cat in a photo, it needs the photo in a form math can touch — a grid of numbers.

The Pixel

An image is a grid of tiny colored squares called pixels. Each pixel carries an intensity value from 0–255 per color channel it has.

Three things fully describe an image:

  • Size — height × width, in pixels
  • Color space — Grayscale, RGB, HSV, etc.
  • Channels — how many numbers each pixel needs (1 for grayscale, 3 for RGB)
🧱 Analogy

Picture a Minecraft build viewed from far away — thousands of single-color blocks that only look like a coherent picture once you back away. Each block is a pixel; its "block type" is its intensity value.

Grayscale blocks only come in shades of gray (one dial: how bright). RGB blocks come from mixing three paint tubes — red, green, blue — so each block needs three dials instead of one.

The RGB Color Scheme

Every RGB pixel is a triplet [R, G, B], each ranging 0–255. 0 means "none of that color," 255 means "full intensity."

ColorRGB
Pure Red25500
Pure Green02550
Pure Blue00255

Grayscale is the special case where R = G = B, so it collapses to one number instead of three.

🎨 Analogy

It's exactly like a stage lighting rig with three spotlights — one red, one green, one blue — all aimed at the same spot. Turn all three to full brightness and the spot looks white. Turn all three off and it's pitch black. Grayscale is what happens when someone swaps all three bulbs for the same shade of white and just dims them together.

Images as Arrays — 2D, 3D, 4D

  • Single grayscale image → 2D array, shape (height, width)
  • Batch of grayscale images → 3D array, shape (samples, height, width)
  • Single RGB image → 3D array, shape (height, width, 3)
  • Batch of RGB images → 4D array, shape (samples, height, width, 3)

This is exactly the tensor shape you'll feed into a neural network later in the course.

📚 Analogy

A grayscale image is a single sheet of graph paper with a number in every box. An RGB image is three transparent sheets — red, green, blue — stacked on a lightbox, each with its own numbers. A whole dataset is just a filing cabinet of these stacks, one drawer per photo.

Pixel Normalization

Raw pixel values (0–255) are rescaled to [0, 1] before feeding a neural network, since small, consistent input ranges make gradient-based training faster and more stable.

normalized_pixel = raw_pixel × (1/255)
e.g. 178 × 0.0039 ≈ 0.698
📏 Analogy

It's like converting every measurement in a recipe to "fraction of a cup" instead of a mix of grams, tablespoons, and pinches. Same information, but now every ingredient speaks the same scale — easier for the (neural) chef to follow consistently.

Module 3 · Feature Extraction

Feature Extraction: HOG, SIFT & the Rise of Convolution

A raw pixel grid is mostly noise for a task like "is there a face here?" Feature extraction strips it down to what actually matters.

What Counts as a "Feature"

A feature is any piece of information extractable from an image — shape, edges, texture, color distribution — that helps a downstream task like classification or detection.

Not every task needs every feature. You can often recognize an object from its edges alone, even with color and background stripped away.

🕵️ Analogy

Think of a police sketch artist. They don't need a full-color photograph to produce a recognizable likeness — a black-and-white outline capturing jawline, eyes, and hairline is often enough for someone to say "that's him." Feature extraction is the CV equivalent of that sketch: keep the outline, drop the noise.

Two Classic Hand-Crafted Techniques

HOG (Histogram of Oriented Gradients)SIFT (Scale-Invariant Feature Transform)
Focuses on the structure and shape of an object by measuring gradient direction and magnitude in local regions. Converts image content into local keypoint coordinates that stay stable under rotation, scaling, or lighting changes.
Good at capturing silhouettes — historically strong for pedestrian detection. Good at matching the same object across two different photos of it (different angle, size, lighting).

Shared weakness: both are computationally expensive, mathematically intricate to hand-tune, and don't generalize cleanly across wildly different image types — exactly the gap convolution was built to close.

Why Convolution Took Over

  • It's trainable — instead of hand-designing what pattern to look for, the model learns the best filter values from data.
  • It scales from low-level features (edges, corners) to high-level features (eyes, wheels, faces) simply by stacking layers.
  • 2012's AlexNet — a CNN built on convolution — beat the runner-up in the ImageNet competition by a double-digit margin, pulling the whole field's attention toward learned filters over hand-crafted ones like HOG and SIFT.
🧑‍🍳

HOG and SIFT are like a chef who follows one fixed recipe no matter what's in the fridge. Convolution — especially inside a CNN — is a chef who tastes the dish as they go and adjusts the recipe itself, layer after layer, until it matches what the diners actually want.

Module 4 · Convolution & Edges

The Convolution Operation & Edge Detection

Convolution is the single operation that lets a small matrix of numbers "scan" an image and light up wherever a pattern it's tuned for appears.

What an Edge Actually Is

Mathematically, an edge is a region where pixel intensity changes sharply between neighboring pixels — a steep gradient, not a gradual fade.

Convolution finds these gradients using a small grid of numbers called a kernel (or filter): slide it across the image, multiply overlapping values, sum them, and write the sum into an output grid called the feature map.

🔦 Analogy

Imagine dragging a small stencil over a textured wall in the dark, feeling for a specific bump pattern with your fingertips. Wherever the wall's texture matches your stencil's shape, you feel a strong "click." A convolution kernel is that stencil — it lights up numerically wherever the image matches its pattern.

  • Prewitt — the simplest gradient kernel; needs a separate horizontal version too.
  • Sobel — Prewitt's center row/column weighted ±2 instead of ±1, giving stronger, cleaner edge response.
  • Laplacian — a second-order kernel that catches edges in every direction with a single pass, but is noise-sensitive, so images are usually smoothed first.
🥁

Prewitt is tapping a table lightly to feel for a seam. Sobel is tapping harder in the middle of the same motion — same idea, stronger signal. Laplacian is more like shaking the whole table at once: it reacts to a seam in any direction, but it also reacts to every little wobble (noise), which is why you steady the table first (blur) before using it.

Convolution on Color (RGB) Images

An RGB image is a 3-channel stack, so the kernel must match: a 3×3×3 filter for a 3-channel image. All 27 kernel numbers multiply against their 27 corresponding pixel values, and every product gets summed into one output number.

Using multiple filters at once (say, one for vertical edges, one for horizontal) produces a stack of feature maps — one per filter — which is exactly the "depth" dimension you'll see in CNN layers.

🎛️

Running several filters over the same image is like having several sound engineers each isolate a different instrument from the same recording — one pulls out the bassline, another the vocals, another the drums. Each engineer (filter) hears the same input but reports back a different, specialized track (feature map).

From Hand-Crafted Filters to CNNs

Prewitt, Sobel, and Laplacian are all fixed, hand-designed kernels — someone chose those exact numbers. A Convolutional Neural Network keeps the sliding-and-summing mechanics identical, but treats the kernel's numbers as learnable weights, discovered automatically through forward propagation and backpropagation.

You still choose the kernel size (e.g. 3×3) as a hyperparameter — but the model decides what pattern that kernel should hunt for.

🎯

It's the difference between handing someone a pre-cut stencil and handing them a blank stencil plus thousands of practice walls, letting them sand it into whatever shape turns out to be most useful. The CNN sands its own stencils.

Student Question: Can an Image Have Four Color Channels?

RGB Versus RGBA

Yes, an image can contain four channels, but the fourth channel is usually not another visible color.

The common four-channel format is RGBA:

ChannelMeaning
RRed intensity
GGreen intensity
BBlue intensity
AAlpha, which stores transparency or opacity

An RGBA pixel such as [255, 0, 0, 255] represents fully opaque red. A value such as [255, 0, 0, 100] is still red, but it is partially transparent.

🪟 Transparency Analogy

RGB describes the paint color. Alpha describes how much of the background can be seen through that paint.

A high alpha value behaves like solid paint. A low alpha value behaves more like tinted glass.

Do Neural Networks Use the Alpha Channel?

Most standard computer-vision datasets use either one grayscale channel or three RGB channels. When an image includes alpha, the transparency channel is often removed before training:

image = image[:, :, :3]

This keeps only the red, green, and blue channels. The alpha channel should be retained only when transparency itself carries useful information for the prediction task.

🧳 Useful Baggage

A neural network can carry a fourth channel, but there is no reason to pack it unless it helps with the journey.

If every image is fully opaque, alpha is just an extra suitcase filled with air.

Can Images Have More Than Four Channels?

Yes. A channel is simply another matrix of measurements aligned with the same image.

Image typeTypical channel count
Grayscale1
RGB3
RGBA4
Satellite imageryMany spectral bands
Hyperspectral imageryDozens or hundreds

Satellite channels may include near-infrared or thermal measurements. These are not visible colors, but they provide useful information about vegetation, water, heat, and materials.

🔭 Extra Lenses

Each channel is another lens observing the same scene. RGB uses three human-visible lenses. Satellite systems may add lenses that detect heat or infrared light that human eyes cannot see.

Classroom answer: “Yes, we can use four channels. The fourth channel is usually alpha, which represents transparency, not a fourth color. Neural networks can also use many more channels when each channel contains a useful measurement.”

Model 1: ANN with RGB Images

Step 1 — Read the Dataset Shape

num_classes = labels['Label'].nunique() counts the number of unique target categories. If the dataset contains cats, dogs, and horses, then num_classes = 3.

image_size = x_train_rgb[0].size calculates the total number of values in one RGB image. For a 32 × 32 × 3 image, this is:

32 × 32 × 3 = 3072 input values

shape = x_train_rgb.shape[1:] removes the batch dimension and stores only the shape of one image, such as (32, 32, 3).

📦 Shape Analogy

A dataset tensor is a warehouse full of image boxes. The first number tells us how many boxes there are. The remaining numbers describe the dimensions of one box.

So (5000, 32, 32, 3) means 5,000 images, each with height 32, width 32, and three color channels.

What Each Layer Does

Sequential and Input

model_1 = Sequential() creates a model where layers are arranged one after another.

Input(shape=(shape[0], shape[1], shape[2])) tells Keras the height, width, and number of channels expected for one image.

Flatten

Flatten() converts the image tensor into a vector. For example:

(32, 32, 3) → (3072,)

This allows the image to enter Dense layers, but it also removes the explicit spatial arrangement of neighboring pixels.

🧵 Flatten Analogy

Flattening is like taking a tiled picture off the wall, numbering every tile, and arranging all tiles in one long line.

The colors are still present, but the model no longer has a built-in notion that tile 41 was directly beside tile 42. That is one reason CNNs usually handle images better than ordinary ANNs.

Dense Layers and ReLU

The hidden layers contain 256, 128, 64, and 32 neurons. Each neuron computes a weighted combination:

z = W·x + b
ReLU(z) = max(0, z)

ReLU keeps positive values and turns negative values into zero, allowing the network to learn nonlinear patterns.

he_uniform initializes the weights in a range designed to work well with ReLU activations and helps keep training stable.

🧠 Feature Funnel

The network starts with thousands of raw pixel values and passes them through a narrowing funnel.

Early neurons react to broad combinations of color and intensity. Later neurons retain only the combinations most useful for deciding the class.

Softmax Output

The final layer has one neuron per class:

Dense(num_classes, activation='softmax')

For three classes, the output might be [0.02, 0.95, 0.03]. The model predicts the second class because it has the highest probability.

🗳️ Probability Vote

Softmax turns the network's final scores into a probability ballot. Every class receives a share, and all shares add up to 100%.

Reading model.summary()

What the Summary Shows

model_1.summary() prints a compact blueprint of the network. For every layer, it reports:

  • Layer name and type — for example, flatten or dense_1.
  • Output shape — the shape of the data after it passes through that layer.
  • Param # — the number of trainable weights and biases in that layer.

The word None in an output shape represents the batch size. It is left unspecified so the model can process different batch sizes.

🏗️ Blueprint Analogy

A model summary is the building plan for the neural network. Each row tells us the size of one floor, how many rooms it contains, and how many adjustable parts are installed there.

None means the building can admit different-sized groups of visitors without redesigning the structure.

The Dense-Layer Parameter Formula

General Formula

Parameters = (input units × output units) + output units

The first term counts all connections between the previous layer and the current layer. The second term counts one bias value for every neuron in the current layer.

Every neuron receives one weight from every input, plus its own bias.

☎️ Connection Analogy

Imagine every neuron in one layer has a direct phone line to every neuron in the next layer. Each phone line is a weight.

Each receiving neuron also has one private adjustment dial, its bias. Count all phone lines, then add one dial per receiving neuron.

Trainable and Non-Trainable Parameters

Trainable parameters: 12,626,730 means all weights and biases will be updated by backpropagation.

Non-trainable parameters: 0 means no layer has frozen weights or fixed statistics.

The reported size of approximately 48.17 MB comes from storing the parameters as 32-bit floating-point numbers:

12,626,730 × 4 bytes ≈ 48.17 MiB
🎛️ Adjustable Knobs

Trainable parameters are the model's adjustable knobs. During learning, backpropagation turns these knobs to reduce prediction error.

More knobs can make the model more expressive, but they also increase memory use, training time, and the risk of overfitting.

Important observation: almost all parameters are in the first Dense layer because flattening creates 49,152 inputs. That first connection alone contributes more than 12.5 million parameters. This parameter explosion is one major reason CNNs are usually preferred for images: convolution uses small shared filters instead of connecting every pixel to every neuron.

Compiling the Model

Adam Optimizer

adam = optimizers.Adam() creates the optimization algorithm that updates the network's weights during training.

Adam adjusts the step size for different parameters and usually performs well as a strong default optimizer.

Categorical Cross-Entropy

categorical_crossentropy measures how far the predicted probability distribution is from the correct one-hot encoded label.

For example, if the correct class is Dog:

True label: [0, 1, 0]
Prediction: [0.20, 0.70, 0.10]

metrics=['accuracy'] asks Keras to report the percentage of correctly classified images.

🧭 Training Analogy

The loss function tells the model how wrong it was. Adam decides how the model should adjust its weights. Accuracy keeps the scoreboard.

Loss is the compass, Adam is the steering system, and accuracy is the dashboard gauge.

Why Compare Four Image Representations?

The comparison helps answer four practical questions:

  • Does color provide useful predictive information?
  • Can grayscale reduce complexity while preserving performance?
  • Does Gaussian blur suppress distracting noise?
  • Do Laplacian edges expose object shape more clearly?

We should compare more than training accuracy. The final decision should consider validation accuracy, test accuracy, loss curves, overfitting, and stability across epochs.

🏁 Final Selection

The best model is not automatically the one that memorizes the training set most aggressively.

It is the one that performs well on unseen images without its training curve turning into a victory parade while the validation curve quietly falls into a ditch.
